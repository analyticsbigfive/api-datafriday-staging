# Résolution des problèmes - 16 Janvier 2026

## Problèmes corrigés

### 1. ✅ Table `WeezeventSyncState` manquante
**Erreur:** `The table 'public.WeezeventSyncState' does not exist in the current database`

**Solution:**
```bash
# Synchronisation du schéma Prisma avec la base de données
docker-compose exec -T api-dev npx prisma db push --accept-data-loss
```

### 2. ✅ Configuration Weezevent du tenant
**Erreur:** `Weezevent not enabled for tenant`

**Solution:**
Deux scripts SQL créés dans `/scripts`:
- `enable-weezevent.sql` - Active Weezevent sur le tenant
- `add-weezevent-credentials.sql` - Ajoute les credentials Weezevent

```bash
# Activer Weezevent
cat scripts/enable-weezevent.sql | docker-compose exec -T api-dev sh -c "npx prisma db execute --stdin"

# Ajouter les credentials
cat scripts/add-weezevent-credentials.sql | docker-compose exec -T api-dev sh -c "npx prisma db execute --stdin"
```

### 3. ✅ Dépendances manquantes dans l'API
**Erreur:** `Cannot find module '@nestjs/axios'`

**Solution:**
- Suppression des volumes Docker obsolètes
- Reconstruction complète de l'API
```bash
docker-compose down -v
docker-compose --profile dev up -d --build
```

### 4. ✅ Configuration Frontend/API
**Erreur:** Port API incorrect (3001 au lieu de 3000)

**Solution:**
- Correction dans `docker-compose.yml`: `VITE_API_URL: http://localhost:3000/api/v1`
- Correction dans `frontend/src/views/DashboardView.vue`: fallback URL

### 5. ✅ Persistance de la session (Déconnexion à chaque refresh)
**Erreur:** La session n'était pas persistée, déconnexion à chaque actualisation

**Solution:**
Modifications dans les fichiers suivants:

**frontend/src/lib/supabase.js** - Configuration explicite de la persistance:
```javascript
export const supabase = createClient(
  config.supabase.url,
  config.supabase.anonKey,
  {
    auth: {
      storage: window.localStorage,
      storageKey: 'datafriday-auth',
      autoRefreshToken: true,
      persistSession: true,
      detectSessionInUrl: true
    }
  }
)
```

**frontend/src/stores/auth.js** - Optimisations:
- Ajout d'une vérification pour éviter les checks répétés
- Amélioration de `onAuthStateChange` avec gestion des événements `TOKEN_REFRESHED`
- Logs de débogage pour suivre les changements d'état d'authentification

## Utilisation

### Voir les logs en continu
```bash
# Tous les services
docker-compose logs -f

# Services spécifiques
docker-compose logs -f frontend-dev api-dev

# Avec limite de lignes
docker-compose logs -f --tail=100 api-dev
```

### Migrations Prisma
```bash
# Vérifier l'état
docker-compose exec api-dev npx prisma migrate status

# Synchroniser le schéma
docker-compose exec api-dev npx prisma db push

# Générer le client Prisma
docker-compose exec api-dev npx prisma generate
```

### Redémarrer les services
```bash
# API uniquement
docker-compose restart api-dev

# Frontend uniquement
docker-compose restart frontend-dev

# Tous les services
docker-compose restart
```
